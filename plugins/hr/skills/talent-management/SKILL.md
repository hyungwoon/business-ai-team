# Talent Management Skill

Frameworks for recruiting, performance management, compensation, and organizational development in Korean and global business contexts.

## Recruiting Framework

### Job Description Template

```
직책명 (Job Title):
팀/부서 (Team/Department):
고용 형태 (Employment Type): 정규직 / 계약직 / 인턴
근무지 (Location):
보고 라인 (Reports to):

[회사 소개 — 2-3문장, 미션 + 현재 단계]

역할 개요 (Role Summary):
[이 역할이 존재하는 이유와 핵심 임팩트 — 3-4문장]

주요 업무 (Key Responsibilities):
- [동사로 시작하는 구체적 업무 — 5-7개]

필수 자격 (Required Qualifications):
- [측정 가능한 경험/스킬 — 3-5개]

우대 사항 (Nice to Have):
- [선호 요소 — 3개 이하]

보상 패키지 (Compensation):
- 연봉: [범위 또는 "경력에 따라 협의"]
- 스톡옵션: [있을 경우 규모]
- 복리후생: [핵심 항목 3-5개]
```

### Interview Process Design

**4단계 표준 프로세스:**

| 단계 | 형식 | 목적 | 소요 시간 |
|---|---|---|---|
| 1. 서류 검토 | 비동기 | 기본 자격 필터링 | - |
| 2. 인재 스크리닝 | 30분 화상 | 동기, 커뮤니케이션 평가 | HR |
| 3. 직무 인터뷰 | 1-2시간 | 기술/역량 심층 평가 | 팀장 + 팀원 |
| 4. 임원 면접 | 30-45분 | 문화 적합도, 최종 확인 | C레벨 |

**역량 기반 인터뷰 질문 구조 (STAR):**
- **Situation:** 상황을 설명해주세요
- **Task:** 당신의 역할/목표는 무엇이었나요?
- **Action:** 구체적으로 어떤 행동을 취했나요?
- **Result:** 결과는 어땠나요? 수치로 설명해주세요

**평가 루브릭 (1-5점):**

| 역량 | 1 (미달) | 3 (충족) | 5 (탁월) |
|---|---|---|---|
| 문제 해결 | 지시 없이는 해결 못함 | 독립적으로 해결 | 새로운 접근법 제시 |
| 커뮤니케이션 | 의견 전달 불명확 | 명확하게 전달 | 설득력 있게 영향 |
| 오너십 | 외부 귀인 경향 | 본인 영역 책임 | 조직 전체에 기여 |

## 성과 관리 (Performance Management)

### OKR 작성 가이드

**Objective 작성 원칙:**
- 영감을 주는 방향성 제시 (측정 불필요)
- 분기 단위, 팀이 자율적으로 달성 가능
- 예: "고객이 우리 제품 없이는 살 수 없게 만든다"

**Key Result 작성 원칙:**
- 구체적, 측정 가능, 기간 명시
- Objective당 3-5개
- 0-1.0 스케일로 진행률 측정
- 예: "NPS를 42에서 58로 향상" / "월 MAU 10만 달성"

**흔한 실수:**
- KR에 활동(Activity)을 적는 것 → 결과(Outcome)로 수정
- KR이 너무 쉬움 (100% 달성 예상) → 70% 달성 목표로 설정
- 개인 OKR이 팀 OKR과 연결 안 됨

### 성과 리뷰 사이클

**반기 리뷰 구조:**

```
자기 평가 (Self-Assessment):
1. 이번 반기 가장 자랑스러운 성과는?
2. 목표 대비 달성률은? (OKR 기준)
3. 개선이 필요한 영역은?
4. 다음 반기 집중할 목표는?

상향 피드백 (Upward Feedback):
1. 팀장이 더 해줬으면 하는 것은?
2. 팀장이 그만했으면 하는 것은?
3. 팀장이 계속했으면 하는 것은?

성과 등급 (Performance Rating):
- Exceeds Expectations (상위 10%)
- Meets Expectations (60-70%)
- Developing (20-25%)
- Below Expectations (5% 이하, PIP 대상)
```

## 보상 체계 (Compensation)

### 급여 밴드 설계

```
레벨 구조 예시 (스타트업):
L1: Junior         — ₩30-45M
L2: Mid-level      — ₩45-65M
L3: Senior         — ₩65-90M
L4: Staff/Lead     — ₩90-120M
L5: Principal      — ₩120-160M
L6: Director+      — ₩160M+

밴드 설계 원칙:
- 시장 P50 기준 중간값 설정 (Radford, 원티드 등 벤치마크)
- 밴드 내 범위: 중간값 ±20%
- 밴드 간 겹침 허용 (경력자 승진 시 급여 역전 방지)
```

### 총 보상 패키지 구성

| 항목 | 스타트업 | 중견기업 |
|---|---|---|
| 기본급 | 시장 P50 | 시장 P50-75 |
| 성과급 | 연 0-30% | 연 10-20% |
| 스톡옵션 | 0.01-1% (직급별) | 제한적 |
| 4대보험 | 필수 | 필수 |
| 식대/교통비 | ₩200K/월 | ₩200K/월 |
| 자기계발비 | ₩1-3M/년 | ₩500K-2M/년 |
| 재택/유연근무 | 높음 | 중간 |

## 조직 문화 & 인게이지먼트

### eNPS (Employee Net Promoter Score)

**측정 방법:** "이 회사를 친구/지인에게 일하기 좋은 곳으로 추천하겠습니까?" (0-10점)
- Promoters: 9-10점
- Passives: 7-8점
- Detractors: 0-6점

**eNPS = Promoters% - Detractors%**

| Score | 해석 |
|---|---|
| 50+ | 탁월 |
| 20-49 | 양호 |
| 0-19 | 개선 필요 |
| 0 미만 | 위험 |

### 온보딩 체크리스트 (첫 90일)

**Day 1:**
- [ ] 업무 장비 및 계정 세팅 완료
- [ ] 팀 소개 미팅 (1:1 + 팀 전체)
- [ ] 회사 미션/비전/전략 오리엔테이션

**Week 1:**
- [ ] 주요 협업 도구 교육 (Slack, Notion, JIRA 등)
- [ ] 첫 업무 태스크 배정
- [ ] 팀장과 기대치 정렬 미팅

**Month 1:**
- [ ] 30일 체크인 (팀장 + HR)
- [ ] 첫 성과물 완료
- [ ] 주요 이해관계자 파악 완료

**Month 3:**
- [ ] 90일 리뷰 (성과 + 문화 적합도)
- [ ] 다음 분기 OKR 설정
- [ ] 온보딩 경험 피드백 수집

## 재택/하이브리드 근무 정책 프레임워크

### 근무 형태 정의

| 유형 | 정의 | 적합한 직무 |
|---|---|---|
| 풀 리모트 | 100% 원격 근무 | 개발, 디자인, 콘텐츠 |
| 하이브리드 (코어) | 주 2-3일 오피스 필수 | 대부분의 직무 |
| 하이브리드 (유연) | 월 X일 오피스 권장 | 시니어 개인 기여자 |
| 오피스 우선 | 오피스 기본, 재택 예외 | 영업, 운영, 고객 대면 |

### 정책 설계 체크리스트

**자격 요건:**
- [ ] 수습 기간(3개월) 이후 신청 가능
- [ ] 최근 성과 등급 "Meets Expectations" 이상
- [ ] 팀장 승인 필수
- [ ] 직무 특성상 원격 수행 가능 여부 확인

**장비 및 환경:**
- [ ] 회사 지급 장비 목록 정의 (노트북, 모니터, 키보드)
- [ ] 재택 근무 환경 지원금 (₩500K-1M, 1회성)
- [ ] 인터넷 비용 지원 여부 (월 ₩30-50K)
- [ ] 보안 요구사항 (VPN 필수, 화면 잠금 정책)

**커뮤니케이션 규범:**
```
코어 타임 (Core Hours): 10:00-16:00 (온/오프라인 모두 응답 가능)
비동기 응답 기준: Slack 메시지 4시간 이내, 이메일 24시간 이내
화상 미팅 에티켓: 카메라 ON 권장, 배경 정돈
문서화 우선: 구두 결정은 반드시 Slack/Notion에 기록
```

**성과 측정:**
- 출근 시간이 아닌 결과물로 평가
- 주간 업무 보고 (팀 공유 채널에 금요일 EOD)
- 분기별 재택 근무 적합성 재평가

## DEI 프로그램 설계

### 다양성 지표 (Diversity Metrics)

**측정 항목 (분기별):**

| 지표 | 측정 방법 | 목표 |
|---|---|---|
| 성별 다양성 | 전체/직급별 성비 | [CUSTOMIZE: 목표 비율] |
| 채용 파이프라인 다양성 | 지원자/면접자/합격자 성비 | 지원자 성비 반영 |
| 승진 형평성 | 직급별 승진율 성별/연령별 비교 | 통계적 유의미한 차이 없음 |
| 급여 형평성 | 동일 직급/경력 급여 격차 | ±5% 이내 |
| 이직률 다양성 | 그룹별 이직률 비교 | 그룹 간 차이 10%p 이내 |

### 포용성 설문 (Inclusion Survey)

**반기별 익명 설문 (1-5점):**
```
소속감:
□ 나는 팀에서 환영받는다고 느낀다
□ 내 의견이 회의에서 진지하게 받아들여진다
□ 나의 배경/경험이 팀에 가치를 더한다고 느낀다

공정성:
□ 성과 평가가 공정하게 이루어진다
□ 승진 기회가 모두에게 동등하게 주어진다
□ 팀장이 편견 없이 피드백을 준다

심리적 안전감:
□ 실수를 해도 비난받지 않는다
□ 다른 의견을 자유롭게 말할 수 있다
□ 차별이나 괴롭힘을 경험한 적 없다
```

### 편견 없는 채용 체크리스트

```
채용 공고:
□ 성별 중립적 언어 사용 (예: "그/그녀" 대신 "지원자")
□ 필수 자격 요건 최소화 (Nice-to-have와 명확히 구분)
□ 급여 범위 공개 (투명성)
□ 다양한 채용 채널 활용 (특정 학교/경력 편중 방지)

면접 과정:
□ 구조화된 인터뷰 (모든 지원자에게 동일한 질문)
□ 다양한 면접관 구성 (성별, 연령, 배경)
□ 평가 기준 사전 합의 (면접 후 논의 전)
□ 개인 신상 질문 금지 (결혼, 자녀, 출신 지역)

의사결정:
□ 개별 평가 후 합의 (앵커링 편향 방지)
□ "문화 핏" 대신 "문화 기여" 기준 사용
□ 탈락 이유 문서화 (감사 가능성)
```

### ERG (Employee Resource Group) 구조

```
ERG 설립 요건:
- 최소 구성원: 5명 이상
- 경영진 스폰서: 1명 (C레벨 또는 VP)
- 연간 활동 계획 제출
- 예산 신청 가능 (연 ₩1-5M)

ERG 유형 예시:
- 여성 리더십 그룹
- 워킹맘/워킹대디 그룹
- 장애인 지원 그룹
- 외국인/다문화 그룹
- LGBTQ+ 지원 그룹

ERG 활동 지원:
- 월 1회 미팅 시간 업무 시간 내 허용
- 사내 커뮤니케이션 채널 제공
- 연 2회 전사 공유 세션
```

## 고용주 브랜딩 전략

### EVP (Employee Value Proposition) 개발

```
EVP 구성 요소:
1. 보상 (Compensation): 급여, 복리후생, 스톡옵션
2. 일의 의미 (Purpose): 미션, 임팩트, 사회적 가치
3. 성장 (Growth): 학습 기회, 커리어 경로, 멘토링
4. 문화 (Culture): 팀 분위기, 일하는 방식, 가치관
5. 환경 (Environment): 근무 유연성, 오피스, 도구

EVP 개발 프로세스:
1. 현직자 인터뷰 (10-15명, 직급/부서 다양하게)
2. 이직자 인터뷰 (퇴사 후 3-6개월 경과자)
3. 경쟁사 EVP 분석 (Glassdoor, LinkedIn 리뷰)
4. EVP 초안 작성 → 현직자 검증
5. 채용 채널별 메시지 커스터마이징
```

### Glassdoor 관리

```
리뷰 관리 원칙:
- 모든 리뷰에 공식 답변 (긍정/부정 모두)
- 부정 리뷰 답변 구조:
  1. 피드백 감사 인사
  2. 구체적 개선 노력 언급
  3. 추가 대화 채널 안내 (HR 이메일)
- 리뷰 조작 절대 금지 (직원에게 리뷰 요청 시 자발성 강조)

모니터링 주기: 월 1회
목표 평점: 3.8+ (업계 평균 3.5)
```

### 캠퍼스 채용 전략

```
타겟 학교 선정 기준:
- 직무 관련 전공 강점
- 과거 채용 성과 (합격률, 재직 기간)
- 지리적 접근성

캠퍼스 채용 타임라인 (상반기 기준):
- 9-10월: 채용 설명회, 부스 참가
- 11월: 서류 접수
- 12월: 1차 면접 (화상)
- 1월: 최종 면접 + 오퍼
- 2-3월: 입사 전 인턴십 또는 사전 교육

인턴십 → 정규직 전환:
- 인턴 기간: 8-12주
- 전환 평가: 프로젝트 발표 + 팀장 평가
- 목표 전환율: 60-70%
```

## 승계 계획 (Succession Planning)

### 핵심 직위 식별

```
핵심 직위 기준 (다음 중 2개 이상 해당):
□ 해당 직위 공석 시 사업 운영에 즉각적 영향
□ 외부 채용 리드타임 6개월 이상
□ 고도의 내부 지식/관계 필요
□ 팀 규모 5명 이상 관리

핵심 직위 목록 (연 1회 업데이트):
| 직위 | 현 담당자 | 이탈 리스크 | 후계자 준비도 |
|---|---|---|---|
| [CUSTOMIZE] | [이름] | 높음/중간/낮음 | 즉시/1년/2년+ |
```

### 준비도 평가 (Readiness Assessment)

**9-Box Grid:**

```
         성과 (Performance)
         낮음    중간    높음
잠재력  높음 | 수수께끼 | 미래 스타 | 일관된 스타 |
(Potential) 중간 | 딜레마  | 핵심 인재 | 고성과자   |
         낮음 | 저성과자 | 효과적 직원| 전문가     |
```

- **일관된 스타 (Consistent Star):** 즉시 승계 가능 후보
- **미래 스타 (Future Star):** 1-2년 내 준비 가능
- **핵심 인재 (Core Player):** 현 직위 유지, 전문성 심화

### 개발 계획 템플릿

```
후계자 개발 계획:

대상자: [이름]
```

## 퇴직 인터뷰 (Exit Interview)

### 구조화된 퇴직 인터뷰 질문

**진행 방식:** HR 담당자가 퇴직 2주 전 진행 (팀장 제외), 익명 처리 후 분석

```
퇴직 사유:
1. 이직을 결심하게 된 주된 이유는 무엇인가요?
2. 이직 결정에 영향을 준 요소를 중요도 순으로 말씀해주세요
   (보상 / 성장 기회 / 팀 문화 / 업무 내용 / 리더십 / 워라밸 / 기타)
3. 어떤 변화가 있었다면 남아있었을까요?

재직 경험:
4. 재직 중 가장 만족스러웠던 것은?
5. 가장 불만족스러웠던 것은?
6. 팀장/리더십에 대한 솔직한 피드백을 주신다면?

조직 개선:
7. 회사가 즉시 개선해야 할 것 1가지는?
8. 이 회사의 강점은 무엇이라고 생각하시나요?
9. 지인에게 이 회사를 추천하시겠습니까? (0-10점)

마무리:
10. 추가로 공유하고 싶은 것이 있으신가요?
```

### 퇴직 데이터 분석 프레임워크

```
분기별 퇴직 분석 리포트:

1. 이직률 현황
   - 전체 이직률 = (퇴직자 수 / 평균 재직자 수) × 100
   - 자발적 이직률 (Voluntary Turnover)
   - 비자발적 이직률 (Involuntary Turnover)
   - 후회되는 이직 (Regrettable Attrition): 고성과자 이탈

2. 세그먼트 분석
   - 직급별 이직률
   - 재직 기간별 이직률 (1년 미만, 1-3년, 3년+)
   - 부서별 이직률

3. 이직 사유 분류
   - 보상 관련: [%]
   - 성장 기회: [%]
   - 리더십: [%]
   - 문화: [%]
   - 외부 요인 (이민, 창업 등): [%]

4. 액션 아이템
   - 이직률 높은 부서 집중 관리
   - 반복되는 이직 사유 → 구조적 개선
```

## 한국 노동법 컴플라이언스

### 근로기준법 핵심 체크리스트

**근로시간 (52시간제):**
```
□ 법정 근로시간: 주 40시간 (1일 8시간)
□ 연장 근로: 주 12시간 한도 (합계 52시간)
□ 연장/야간/휴일 근로 가산수당: 통상임금의 50%
□ 포괄임금제 사용 시: 실제 연장 근로 시간 초과 여부 모니터링
□ 근로시간 기록 의무: 출퇴근 기록 2년 보관
□ 특례 업종 여부 확인 (운수, 보건 등 일부 업종 예외)
```

**연차유급휴가:**
```
□ 1년 미만: 1개월 개근 시 1일 (최대 11일)
□ 1년 이상: 15일 (3년마다 1일 추가, 최대 25일)
□ 연차 사용 촉진 제도 활용 시 미사용 연차 수당 면제 가능
□ 연차 사용 촉진 절차: 10일 전 서면 통보 → 5일 전 재통보
□ 연차 수당 지급 기준: 통상임금 기준
```

**퇴직금:**
```
□ 지급 요건: 1년 이상 계속 근로 + 주 15시간 이상
□ 지급 기준: 계속 근로 1년에 대해 30일분 평균임금
□ 지급 시기: 퇴직일로부터 14일 이내
□ IRP(개인형 퇴직연금) 계좌로 지급 의무 (2022년 이후)
□ 중간 정산: 원칙적 금지 (주택 구입 등 예외 사유 한정)
```

**비정규직 보호법:**
```
□ 기간제 근로자: 2년 초과 사용 시 무기계약직 전환 의무
□ 파견 근로자: 2년 초과 사용 시 직접 고용 의무
□ 차별 금지: 동종/유사 업무 정규직 대비 불합리한 차별 금지
□ 기간제 계약 갱신 시: 갱신 기대권 발생 여부 검토
```

**직장 내 괴롭힘 방지법:**
```
□ 신고 접수 창구 운영 의무 (HR 또는 외부 기관)
□ 신고 접수 후 조사 의무 (지체 없이)
□ 피해자 보호 조치: 근무 장소 변경, 유급 휴가 등
□ 가해자 징계 또는 근무 장소 변경
□ 신고자 불이익 처우 금지 (위반 시 3년 이하 징역)
□ 연 1회 예방 교육 의무
□ 취업규칙에 괴롭힘 예방/처리 절차 명시
```

## PIP (Performance Improvement Plan)

### PIP 설계 원칙

PIP는 퇴직 유도 수단이 아니라 실질적 성과 개선 기회여야 한다. 한국 노동법상 해고는 "정당한 이유"가 필요하며, PIP 없이 성과 부진을 이유로 해고 시 부당해고 분쟁 위험이 높다.

### PIP 템플릿

```markdown
# 성과 개선 계획 (Performance Improvement Plan)

**대상자:** [이름]
**직책:** [직책]
**팀장:** [이름]
**HR 담당자:** [이름]
**PIP 기간:** YYYY-MM-DD ~ YYYY-MM-DD (통상 60-90일)
**작성일:** YYYY-MM-DD

## 현재 성과 문제 (Performance Issues)
[구체적 사실 기반으로 서술. 날짜, 수치, 구체적 사례 포함]

예시:
- 2026년 1월 스프린트: 할당된 5개 태스크 중 2개만 완료 (목표: 4개 이상)
- 코드리뷰 응답 시간: 평균 3일 (팀 기준: 1일 이내)
- 3회 연속 데드라인 미준수 (1/5, 1/19, 2/2)

## 개선 목표 (Improvement Goals)
[SMART 기준: 구체적, 측정 가능, 달성 가능, 관련성, 기한]

| 목표 | 측정 기준 | 기한 |
|---|---|---|
| [CUSTOMIZE] | [CUSTOMIZE] | YYYY-MM-DD |

## 지원 계획 (Support Plan)
- 팀장 주간 1:1: 매주 [요일] [시간]
- 멘토 배정: [이름]
- 교육/리소스: [CUSTOMIZE]
- 업무 조정: [CUSTOMIZE: 필요 시 업무량 조정]

## 체크인 일정 (Check-in Schedule)
- 2주차 중간 점검: YYYY-MM-DD
- 4주차 중간 점검: YYYY-MM-DD
- 최종 평가: YYYY-MM-DD

## 결과 시나리오 (Outcomes)
- 목표 달성 시: PIP 종료, 정상 업무 복귀
- 부분 달성 시: PIP 연장 또는 직무 재배치 검토
- 미달성 시: 추가 인사 조치 (직무 변경, 권고사직 등)

## 서명
대상자: _________________ 날짜: _______
팀장: _________________ 날짜: _______
HR: _________________ 날짜: _______
```

### 한국 법적 고려사항

```
PIP 진행 시 필수 준수 사항:

□ 서면 통보 및 서명 수령 (구두 통보 불인정)
□ 개선 기회 충분히 부여 (최소 60일 권장)
□ 구체적 개선 목표 명시 (모호한 기준 금지)
□ 지원 계획 포함 (회사의 개선 노력 입증)
□ 중간 피드백 기록 보관
□ 최종 평가 결과 서면 통보

해고 시 추가 요건:
□ 해고 예고: 30일 전 서면 통보 또는 30일분 통상임금 지급
□ 해고 사유 서면 통보 의무
□ 부당해고 구제 신청 기간: 해고일로부터 3개월 이내
□ 5인 미만 사업장: 근로기준법 해고 규정 일부 미적용 (확인 필요)
```
