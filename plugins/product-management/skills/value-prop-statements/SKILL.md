# 가치 제안 문장

기존 가치 제안으로부터 마케팅, 영업, 온보딩용 가치 제안 문장을 생성합니다. 타깃 세그먼트를 다루고, 혜택을 강조하며, 역량을 부각하는 문장을 만들어냅니다. 타깃 마케팅 콘텐츠, 영업 프레젠테이션, 고객 온보딩 메시지 작성에 최적입니다.

## 사용 시점

- 마케팅 카피 및 프로모션 콘텐츠 작성 시
- 영업 덱 및 피치 자료 제작 시
- 고객 온보딩 메시지 작성 시
- 세그먼트별 메시지 개발 시
- 트리거 키워드: value proposition statements, marketing copy, sales messaging, value statements, positioning copy

## 프롬프트

당신은 가치 제안 개발과 타깃 메시징 분야의 전문 제품 성장 전문가입니다.

$ARGUMENTS에 대한 다음 가치 제안을 바탕으로, 마케팅, 영업, 온보딩 맥락 전반에서 활용할 수 있는 포괄적인 가치 제안 문장을 개발하세요.

각 문장은 다음을 충족해야 합니다:
- 특정 타깃 시장 세그먼트 또는 사용 사례를 직접 다룰 것
- 주요 혜택과 기대 결과를 강조할 것
- 이를 가능하게 하는 핵심 기능과 역량을 부각할 것
- 고객에게 공감을 불러일으키는 명확하고 설득력 있는 언어를 사용할 것

## 예시 프레임워크 (Canva)

접근 방식을 설명하기 위해 Canva의 가치 제안 문장 예시를 제시합니다:

1. **소셜 미디어 마케터를 위한**: Canva는 소셜 미디어 마케터가 값비싼 디자인 소프트웨어나 전담 디자이너 없이도 멋진 브랜드 디자인을 손쉽게 만들 수 있도록 지원합니다. 모든 채널에서 참여율을 높이고 브랜드 일관성을 강화하는 전문가 수준의 그래픽을 신속하게 제작하세요.

2. **소규모 사업주를 위한**: Canva의 직관적인 드래그 앤 드롭 인터페이스와 방대한 사전 디자인 템플릿 컬렉션을 통해 소규모 사업주는 몇 분 만에 세련된 마케팅 캠페인을 시작할 수 있습니다. 사전 디자인 경험 없이도 전문가가 디자인한 것처럼 보이는 웹사이트 그래픽, 소셜 포스트, 전단지, 홍보 자료를 만드세요.

3. **콘텐츠 크리에이터를 위한**: Canva를 사용하면 콘텐츠 크리에이터가 디자인 작업에 시간을 덜 쏟고 스토리텔링에 집중할 수 있습니다. 다양한 플랫폼에 맞춘 템플릿으로 일관되고 시각적으로 매력적인 콘텐츠를 대규모로 제작하여 오디언스 참여와 콘텐츠 전략에 더 많은 시간을 할애하세요.

## Value Proposition Canvas

Alexander Osterwalder의 프레임워크로 고객 관점과 제품 관점을 체계적으로 매칭합니다:

### 고객 프로필 (Customer Profile)

| 구성 요소 | 질문 | 작성 가이드 |
|-----------|------|------------|
| **Jobs (해야 할 일)** | 고객이 달성하려는 기능적·사회적·감성적 작업은? | 동사로 시작, 구체적 상황 포함 |
| **Pains (고충)** | 작업 수행 전·중·후에 겪는 불편과 리스크는? | 심각도 순으로 나열 |
| **Gains (기대 이익)** | 고객이 원하는 결과와 혜택은? | 필수 vs 기대 vs 놀라움 구분 |

### 가치 맵 (Value Map)

| 구성 요소 | 질문 | 작성 가이드 |
|-----------|------|------------|
| **Pain Relievers** | 어떤 고충을 해소하는가? | 상위 3개 고충에 집중 |
| **Gain Creators** | 어떤 기대 이익을 만들어내는가? | 차별화된 이익 강조 |
| **Products & Services** | 어떤 제품·기능으로 전달하는가? | 핵심 기능만 나열 |

활용 시점: 가치 제안을 처음 설계할 때, 또는 기존 가치 제안이 고객 관점과 괴리가 있을 때. Pains와 Pain Relievers, Gains와 Gain Creators가 1:1로 매칭되는지 확인하세요.

## 메시징 계층 구조 (Messaging Hierarchy)

가치 제안을 실제 마케팅 카피로 변환하는 3단계 구조:

- **H1 (헤드라인)**: 핵심 가치를 한 문장으로. 고객의 최우선 니즈를 직접 언급 (예: "팀의 모든 작업을 하나로")
- **H2 (서브헤드라인)**: H1을 구체화하는 방법론 또는 차별점 (예: "문서, 프로젝트, 위키를 하나의 워크스페이스에서")
- **Supporting (지지 메시지)**: 3-4개의 구체적 혜택 포인트. 각 포인트는 기능 → 혜택 구조로 작성

### 채널별 적용 가이드

| 채널 | H1 길이 | H2 포함 | Supporting |
|------|---------|---------|------------|
| 랜딩 페이지 | 8-12단어 | 필수 | 3-4개 섹션 |
| 광고 카피 | 5-8단어 | 선택 | 1-2줄 |
| 영업 덱 | 10-15단어 | 필수 | 슬라이드별 1개 |
| 온보딩 이메일 | 6-10단어 | 필수 | 2-3개 불릿 |

## A/B 테스트 템플릿

가치 제안 문장의 효과를 검증하기 위한 테스트 설계:

| 항목 | 내용 |
|------|------|
| **가설** | "[변형 B]가 [변형 A]보다 [지표]를 [X%] 개선할 것이다" |
| **테스트 변수** | 헤드라인 / 서브헤드라인 / CTA 문구 (한 번에 하나만) |
| **성공 지표** | CTR, 전환율, 가입률 등 |
| **최소 샘플 크기** | 통계적 유의성을 위해 변형당 최소 200-500 전환 |
| **테스트 기간** | 최소 1-2주 (요일별 편차 반영) |

## 안티패턴

- **기능 나열형**: "AI 기반, 실시간, 클라우드 네이티브" → 기능이 아닌 고객 혜택으로 변환할 것
- **모두를 위한 제품**: "모든 비즈니스를 위한" → 특정 세그먼트를 명시해야 공감을 얻음
- **경쟁사 폄하**: "X와 달리 우리는~" → 경쟁사 언급 없이 자체 가치를 전달할 것
- **내부 언어 사용**: "마이크로서비스 아키텍처" → 고객이 이해하는 언어로 번역할 것
- **측정 불가능한 주장**: "최고의 솔루션" → 구체적 숫자나 사례로 뒷받침할 것

## 최상의 결과를 위한 팁

- 기존 가치 제안 또는 핵심 혜택을 제공하세요
- 타깃 세그먼트와 그들의 페인 포인트를 명시하세요
- 제품 기능과 차별점을 포함하세요
- 배포 채널(마케팅, 영업, 온보딩)을 공유하세요
- 브랜드 톤 또는 보이스 가이드라인이 있다면 언급하세요
