## 고객 인터뷰 요약

인터뷰 트랜스크립트를 Jobs to Be Done, 만족도, 액션 아이템 중심의 구조화된 요약으로 변환합니다.

### 컨텍스트

**$ARGUMENTS**의 제품 디스커버리를 위한 고객 인터뷰를 요약합니다.

사용자가 인터뷰 트랜스크립트를 제공합니다 — 첨부 파일(텍스트, PDF, 음성 전사본) 또는 직접 붙여넣기 방식으로. 첨부 파일이 있다면 먼저 읽으세요.

### 지시문

1. **전체 트랜스크립트를 꼼꼼히 읽은 후** 요약하세요.

2. **아래 요약 템플릿을 채우세요.** 정보가 없는 경우 "-"를 사용하세요. 필요 시 숫자 값을 정성적 표현으로 대체하세요 (예: "만족하지 않음").

3. **명확하고 쉬운 언어를 사용하세요** — 초등학생도 이해할 수 있는 수준으로 작성하세요.

### 출력 템플릿

```
**날짜**: [인터뷰 날짜 및 시간]
**참여자**: [전체 이름 및 역할]
**배경**: [고객에 대한 배경 정보]

**현재 솔루션**: [현재 사용 중인 솔루션]

**현재 솔루션에서 좋아하는 점**:
- [수행할 작업(Job to be done), 원하는 결과, 중요도, 만족도 수준]

**현재 솔루션의 문제점**:
- [수행할 작업(Job to be done), 원하는 결과, 중요도, 만족도 수준]

**주요 인사이트**:
- [예상치 못한 발견 또는 주목할 만한 인용구]

**액션 아이템**:
- [날짜, 담당자, 액션 — 예: "2025-01-15, Paweł Huryn, 가격 관련 고객 후속 연락"]
```

요약본을 사용자의 워크스페이스에 마크다운 문서로 저장하세요.

## Affinity Mapping 프로세스

여러 인터뷰를 종합할 때 사용하는 테마 발견 기법:

1. **추출**: 각 인터뷰에서 핵심 발언·관찰을 개별 항목으로 분리 (1항목 = 1인사이트)
2. **그룹핑**: 유사한 항목끼리 묶어 클러스터 형성 (상향식, 미리 카테고리를 정하지 않음)
3. **라벨링**: 각 클러스터에 테마 이름 부여 (예: "온보딩 마찰", "가격 민감도")
4. **우선순위화**: 빈도(몇 명이 언급했는가)와 강도(얼마나 강하게 표현했는가)로 테마 순위 산정
5. **시사점 도출**: 상위 테마에서 제품 기회 또는 위험 요소 도출

활용 시점: 3건 이상의 인터뷰를 종합해야 할 때. 개별 인터뷰 요약 후 이 프로세스로 패턴을 발견하세요.

## Interview Synthesis Matrix

테마와 참여자를 교차 분석하는 매트릭스:

| 테마 | 참여자 A | 참여자 B | 참여자 C | 빈도 | 강도 |
|------|---------|---------|---------|------|------|
| [테마 1] | ✓ "직접 인용" | ✓ | - | 2/3 | 높음 |
| [테마 2] | - | ✓ "직접 인용" | ✓ | 2/3 | 중간 |
| [테마 3] | ✓ | ✓ | ✓ | 3/3 | 높음 |

- **빈도**: 전체 참여자 중 언급한 비율 (3/5 이상이면 강한 패턴)
- **강도**: 감정적 표현 정도 (높음 = "정말 불편", 중간 = "약간 아쉬운", 낮음 = "괜찮긴 한데")

## 인사이트 계층 구조

관찰에서 실행 가능한 결론까지 4단계로 구조화:

1. **관찰 (Observation)**: 인터뷰에서 실제로 보고 들은 것. 해석 없이 사실만 기록
   - 예: "5명 중 4명이 온보딩 3단계에서 이탈했다고 언급"
2. **인사이트 (Insight)**: 관찰에서 도출한 패턴과 의미. "왜"에 대한 해석
   - 예: "온보딩 3단계의 정보 입력량이 사용자의 인지 부하를 초과한다"
3. **시사점 (Implication)**: 인사이트가 제품에 의미하는 바
   - 예: "온보딩 완료율이 낮아 활성화 지표에 부정적 영향"
4. **권고사항 (Recommendation)**: 구체적 행동 제안
   - 예: "온보딩 3단계를 2개 하위 단계로 분할하고, 선택 입력 항목을 제거"

## 종합 보고서 산출물 템플릿

```
## 인터뷰 종합 보고서 — [프로젝트명] [날짜]

### 개요
- 인터뷰 수: [N]건
- 기간: [시작일] ~ [종료일]
- 참여자 프로필: [역할/세그먼트 요약]

### 핵심 테마 (우선순위순)
1. **[테마명]** — 빈도: [X/N], 강도: [높음/중간/낮음]
   - 관찰: [요약]
   - 인사이트: [해석]
   - 권고사항: [액션]

### 테마별 주요 인용구
- "[직접 인용]" — 참여자 [ID], [역할]

### 다음 단계
- [ ] [권고사항 기반 액션 아이템]
```

## 실무 팁

- 인터뷰 직후 24시간 내에 요약을 작성해야 맥락과 뉘앙스를 보존할 수 있음
- 직접 인용구를 반드시 포함 — 2차 정보보다 참여자의 원문이 이해관계자를 설득하는 데 강력
- 참여자 이름 대신 ID(P1, P2)를 사용하여 개인정보 보호
- 인사이트와 관찰을 분리해야 함 — 관찰에 해석이 섞이면 편향이 발생
