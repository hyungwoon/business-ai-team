## 린 스타트업 실험 설계 (신규 제품)

최소한의 노력으로 새로운 제품 컨셉을 검증하기 위한 XYZ 가설을 수립하고 프리토타입 실험을 설계합니다.

### 맥락

린 스타트업 방법론을 활용하여 신규 제품 컨셉을 검증하도록 지원합니다: **$ARGUMENTS**.

사용자가 파일(시장 조사, 랜딩 페이지 목업)을 제공하면 먼저 읽습니다.

### 지시 사항

1. **XYZ 가설을 다음 형식으로 수립**: "최소 X%의 Y가 Z를 할 것이다"
   - **X%**: 참여할 것으로 예상되는 목표 시장의 비율
   - **Y**: 구체적인 목표 시장 (예: "중형 럭셔리 세단 구매자")
   - **Z**: 제품과 어떻게 상호작용할 것인지

2. **최소한의 노력으로 가설을 테스트하기 위한 2-3개의 프리토타입 실험 제안**. 다음을 고려합니다:
   - **랜딩 페이지**: 가입이나 클릭을 측정하여 관심도 테스트
   - **설명 영상**: 참여 지표를 통해 이해도와 매력도 테스트
   - **이메일 캠페인**: 응답률과 클릭률을 통해 수요 테스트
   - **사전 주문 / 대기자 명단**: 실질적인 약속을 통해 구매 의향 테스트
   - **컨시어지 / 수동 MVP**: 서비스를 수동으로 제공하여 가치 테스트

3. **핵심 원칙** (Alberto Savoia, *The Right It*):
   - **스킨-인-더-게임(Skin-in-the-Game)**: 단순한 관심이 아닌 구매 의향을 테스트합니다. 실질적인 약속(시간, 돈, 명성)만이 신뢰할 수 있는 신호입니다.
   - **자체 데이터(YODA, Your Own Data)**: 시장 보고서나 유사 사례 같은 타인의 데이터(ODP)에 의존하지 않고 실험을 통해 자체 데이터를 수집합니다. "당신의 아이디어를 위한 시장은 다른 사람의 아이디어를 위한 시장에 관심이 없습니다."
   - 사용자의 의견이 아닌 실제 행동을 측정합니다

4. **각 실험에 대해** 검증하는 가설, 방법, 지표, 성공 임계값을 명시합니다.

단계별로 생각합니다. 내용이 많으면 마크다운으로 저장합니다.

## 실험 설계 프레임워크

### Lean Experiment Board

신규 제품 실험을 체계적으로 설계하고 추적하는 보드:

| 항목 | 내용 |
|------|------|
| **가설** | "최소 X%의 Y가 Z를 할 것이다" |
| **가장 위험한 가정** | [핵심 가정 1개] |
| **실험 방법** | [프리토타입 유형] |
| **측정 지표** | [행동 기반 지표] |
| **성공 기준** | [구체적 수치] |
| **소요 기간/비용** | [예상 기간, 예상 비용] |
| **결과** | [실험 후 기록] |
| **다음 행동** | Pivot / Persevere / Zoom-in / Zoom-out |

활용 시점: XYZ 가설 수립 직후, 구체적 실험을 설계할 때 사용합니다. 한 가설당 하나의 보드를 작성하여 실험의 전체 흐름을 관리합니다.

### MVP 실험 유형 분류

신규 제품 검증에 적합한 프리토타입 유형별 특성:

| 실험 유형 | 검증 대상 | 비용 | 기간 | 신뢰도 |
|-----------|-----------|------|------|--------|
| **랜딩 페이지** | 시장 관심도, 메시지 공명 | 낮음 | 1-3일 | 중간 |
| **설명 영상** | 컨셉 이해도, 매력도 | 낮음 | 3-5일 | 중간 |
| **컨시어지 MVP** | 핵심 가치, 지불 의향 | 중간 | 1-4주 | 높음 |
| **오즈의 마법사** | 사용자 행동, 워크플로우 | 중간 | 1-2주 | 높음 |
| **스모크 테스트** | 구매 의향, 수요 규모 | 낮음 | 1-2주 | 높음 |
| **사전 주문** | 실질적 구매 의사 | 낮음 | 1-2주 | 매우 높음 |
| **크라우드펀딩** | 시장 수요, 가격 검증 | 중간 | 4-6주 | 매우 높음 |

활용 시점: 가설의 위험 유형에 따라 적합한 실험을 선택합니다. 수요 검증에는 스모크 테스트/사전 주문, 가치 검증에는 컨시어지/오즈의 마법사가 효과적입니다.

### 실험 설계 체크리스트

실험 시작 전 반드시 확인할 항목:

- [ ] 검증할 가정이 하나로 명확히 정의되었는가?
- [ ] 성공/실패 기준이 실험 시작 전에 합의되었는가?
- [ ] 측정 지표가 의견이 아닌 행동 기반인가?
- [ ] Skin-in-the-Game 요소가 포함되었는가? (시간, 돈, 명성 등)
- [ ] 타인의 데이터(ODP)가 아닌 자체 데이터(YODA)를 수집하는가?
- [ ] 최소한의 비용과 시간으로 설계되었는가?
- [ ] 실험 결과에 따른 다음 행동이 미리 정의되었는가?
