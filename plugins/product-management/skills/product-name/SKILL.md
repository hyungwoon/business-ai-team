# 제품명

브랜드 가치와 타깃 고객에 부합하는 독창적이고 기억에 남는 제품명과 근거를 브레인스토밍합니다. 신제품 네이밍, 리브랜딩, 또는 브랜드 포지셔닝을 강화하는 네이밍 옵션 탐색 시 사용하세요.

## 사용 시점

- 신제품 또는 기능 네이밍 시
- 기존 제품 리브랜딩 또는 개명 시
- 출시 전 네이밍 옵션 탐색 시
- 브랜드 가이드라인에 맞게 이름을 검증할 때
- 트리거 키워드: product name, name ideas, brand name, naming, what to call, product naming

## 프롬프트

당신은 제품 네이밍, 브랜드 아키텍처, 시장 포지셔닝 분야의 전문 브랜딩 컨설턴트입니다.

다음 회사 및 제품 맥락을 바탕으로: $ARGUMENTS

회사의 브랜드 가치, 타깃 고객, 시장 포지셔닝에 부합하는 독창적이고 기억에 남는 제품명 5가지를 제안하세요.

각 네이밍 제안에 대해 다음을 제시하세요:

1. **이름**: 제안하는 제품명
2. **근거**: 이 이름이 효과적인 이유—제품의 가치를 어떻게 반영하는지, 타깃 고객에게 어떻게 어필하는지, 브랜드 포지셔닝과 어떻게 부합하는지 설명하세요
3. **브랜드 적합성**: 이 이름이 전반적인 브랜드 아키텍처와 메시징 전략을 어떻게 지원하는지
4. **기억 용이성**: 이 이름이 독특하고, 기억하기 쉬우며, 시장에서 차별화되는 이유
5. **도메인 및 상표 고려사항**: 가용성과 잠재적 상표/도메인 문제에 대한 간략한 메모

다음 조건을 충족하는 이름을 우선시하세요:
- 발음하고 철자를 쓰기 쉬울 것
- 경쟁사와 독특하게 차별화될 것
- 브랜드 톤 및 포지셔닝과 부합할 것
- 제품의 핵심 가치와 사용 사례에 관련될 것
- 상표 및 도메인 등록이 가능할 것

## 네이밍 아키타입 (Naming Archetypes)

이름을 발상할 때 다음 5가지 아키타입 중 하나 이상을 기반으로 접근하세요:

- **서술형 (Descriptive)**: 제품 기능을 직접 설명 (예: YouTube, Salesforce). 활용 시점: 카테고리가 새롭거나 기능 전달이 핵심일 때
- **두문자형 (Acronym)**: 핵심 단어의 이니셜 조합 (예: AWS, SAP). 활용 시점: B2B/엔터프라이즈 시장, 이미 긴 정식 명칭이 있을 때
- **은유형 (Metaphor)**: 추상적 이미지를 차용 (예: Slack, Apple). 활용 시점: 감성적 브랜딩과 차별화가 중요할 때
- **조어형 (Invented)**: 새로 만든 단어 (예: Spotify, Kodak). 활용 시점: 도메인 확보와 강력한 상표권이 필요할 때
- **창업자형 (Founder)**: 사람 이름 기반 (예: Bloomberg, Tesla). 활용 시점: 개인 전문성이 브랜드 핵심 자산일 때

## 네이밍 평가 매트릭스

후보 이름을 다음 기준으로 평가하세요 (각 1-5점):

| 평가 기준 | 설명 | 가중치 |
|-----------|------|--------|
| **기억 용이성** | 한 번 듣고 기억할 수 있는가? | 높음 |
| **발음 용이성** | 타깃 시장 언어로 자연스럽게 발음되는가? | 높음 |
| **도메인 가용성** | .com 또는 주요 도메인 확보 가능한가? | 중간 |
| **상표 리스크** | 기존 상표와 충돌 가능성은? | 높음 |
| **의미 적합성** | 제품 가치와 브랜드 방향에 부합하는가? | 중간 |
| **확장성** | 제품 라인 확장 시에도 유효한가? | 낮음 |
| **문화적 적합성** | 타깃 시장에서 부정적 의미가 없는가? | 높음 |

## 브랜드 네이밍 프로세스 템플릿

단계별 워크시트:

1. **브리프 정의**: 제품 핵심 가치 3개, 타깃 고객, 톤 앤 매너, 경쟁사 이름 목록 정리
2. **발산 (Diverge)**: 아키타입별로 최소 20개 후보 생성 (양 > 질)
3. **1차 필터**: 발음·기억·의미 기준으로 10개로 축소
4. **2차 필터**: 도메인 가용성 + 상표 검색으로 5개로 축소
5. **검증**: 타깃 고객 3-5명에게 후보를 보여주고 첫인상 수집
6. **최종 선정**: 평가 매트릭스 점수 + 고객 피드백 종합하여 결정

## 실무 팁

- 네이밍 회의 전 경쟁사 이름을 칠판에 적어두면 무의식적 유사성을 방지할 수 있음
- 2음절 이름이 기억과 구전에 가장 유리 (Slack, Zoom, Notion)
- 글로벌 출시 예정이면 주요 타깃 시장 언어로 발음·의미를 반드시 교차 검증
- .com 도메인이 불가하면 get[이름].com, [이름]app.com 등 대안 패턴 검토
- 내부 코드네임과 최종 제품명을 구분 — 코드네임에 감정적 애착이 생기면 객관적 판단이 어려움

## 최상의 결과를 위한 팁

- 브랜드 가이드라인과 톤 오브 보이스를 공유하세요
- 타깃 고객과 그들의 선호도를 명시하세요
- 경쟁사 이름과 차별화하고 싶은 부분을 언급하세요
- 회사에서 사용하는 네이밍 규칙 또는 패턴이 있다면 포함하세요
- 제품의 핵심 가치 제안과 주요 기능을 공유하세요
- 고려해야 할 지역 시장 또는 언어가 있다면 언급하세요
