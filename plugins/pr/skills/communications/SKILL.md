# PR & Communications Skill

Frameworks for media relations, press release writing, crisis communications, and brand reputation management.

## 보도자료 (Press Release) 작성

### 표준 구조

```
[보도자료]                              배포일: 즉시 배포 / 엠바고: YYYY-MM-DD 00:00

[헤드라인 — 핵심 뉴스를 한 문장으로, 70자 이내]

[부제목 — 헤드라인 보완, 컨텍스트 추가 — 선택]

[도시], [날짜] — [회사명]이 오늘 [핵심 발표 내용]을 발표했다.

[리드 단락: 누가, 무엇을, 언제, 어디서, 왜 — 2-3문장]

[본문 단락 1: 발표의 중요성과 배경]

[임원 인용구 — 회사 관점]
"[임원 이름], [직책]은 '[발표에 대한 의미 있는 코멘트]'라고 말했다."

[본문 단락 2: 상세 내용, 기능, 수치]

[고객/파트너 인용구 — 외부 시각 신뢰도 부여]

[본문 단락 3: 산업 컨텍스트, 트렌드]

[향후 계획 또는 Call to Action]

[회사 소개 (Boilerplate) — 3-4문장, 회사 설명 표준 문안]

문의:
홍보팀 | [이름] | [이메일] | [전화번호]
```

### 헤드라인 작성 원칙

- **능동태** 사용: "OO가 출시했다" (수동태 지양)
- **수치 포함** 시 임팩트 강화: "MAU 100만 달성한 OO, 시리즈B 200억 유치"
- **SEO 키워드** 고려: 언론사 검색 노출 최적화
- **클리셰 금지**: "세계 최초", "혁신적인", "게임체인저" → 구체적 사실로 대체

## 미디어 관계 (Media Relations)

### 미디어 리스트 구성

**카테고리별 분류:**

| 미디어 유형 | 국내 예시 | 타겟 스토리 |
|---|---|---|
| 종합 경제지 | 한국경제, 매일경제, 서울경제 | 투자, 성장, M&A |
| IT/테크 전문지 | ZDNet, 디지털타임스, 전자신문 | 제품, 기술, 스타트업 |
| 스타트업 미디어 | 플래텀, 벤처스퀘어, IT조선 | 창업 스토리, 성장 지표 |
| 방송 | MBC 경제, KBS 뉴스, SBS Biz | 사회적 임팩트, 소비자 트렌드 |
| 잡지 | Forbes Korea, Harvard Business Review Korea | 리더십, 전략, 문화 |
| 글로벌 | TechCrunch, Bloomberg, Reuters | 글로벌 확장, 해외 투자 |

**기자 DB 관리 항목:**
- 이름 / 소속 / 담당 섹션
- 이메일 / 전화번호
- 최근 기사 3개 (관심 주제 파악)
- 접촉 히스토리 (날짜, 내용, 결과)
- 관계 강도 (Cold / Warm / Hot)

### 미디어 피칭 (Pitching)

**이메일 피치 구조 (150자 이내 제목 포함):**

```
제목: [기자명] 기자님께 — [간결한 뉴스 앵글 한 줄]

[기자님 이름] 기자님,

최근 [기자의 최근 기사 또는 관심 주제] 기사 잘 읽었습니다.

[회사명]에서 [기자의 독자에게 흥미로울 뉴스]를 준비했습니다.

핵심 포인트:
• [데이터 또는 트렌드]
• [독자 관련성]
• [차별화 요소]

더 자세한 내용과 인터뷰 기회를 제공할 수 있습니다.
관심 있으시면 언제든 연락 주세요.

[이름] 드림
```

**피칭 원칙:**
- 기자 마감 시간 피하기 (일간지: 오후 3-5시 피할 것)
- 독점(Exclusive) 제공 시 반응률 3-5배 상승
- 팔로업은 3일 후 1회만 (스팸 인식 방지)

## 위기 커뮤니케이션 (Crisis Communications)

### 위기 등급 분류

| 등급 | 정의 | 예시 | 대응 시간 |
|---|---|---|---|
| P1 | 사업 존속 위협, 법적 리스크 | 데이터 유출, 인명 사고 | 1-2시간 내 |
| P2 | 브랜드 심각한 손상 가능 | 임원 스캔들, 제품 결함 | 4-8시간 내 |
| P3 | 부정적 언론 보도 | 고객 불만 확산, 경쟁사 공격 | 24시간 내 |
| P4 | 모니터링 필요 | SNS 부정 언급 증가 | 48-72시간 내 |

### 위기 대응 프로세스 (Golden Hour)

**0-1시간:**
- [ ] 위기 등급 결정
- [ ] 위기 대응팀 소집 (CEO, 법무, PR, 운영)
- [ ] 팩트 수집 — 아는 것 / 모르는 것 / 확인 중인 것 분리
- [ ] 내부 커뮤니케이션 먼저 (직원, 이사회)

**1-4시간:**
- [ ] 홀딩 스테이트먼트 배포 ("현재 확인 중이며, 추가 정보 공유 예정")
- [ ] 핵심 이해관계자 직접 연락 (주요 고객, 파트너, 투자자)
- [ ] 소셜미디어 모니터링 강화
- [ ] 미디어 문의 대응 프로토콜 가동

**4-24시간:**
- [ ] 공식 입장문 발표
- [ ] 필요 시 기자회견 또는 인터뷰
- [ ] Q&A 문서 준비 (예상 질문 50개)

### 위기 시 절대 하지 말아야 할 것

- **"노코멘트"** → 은폐로 해석됨. 대신: "현재 확인 중입니다"
- **사실 확인 전 전면 부인** → 나중에 번복 시 신뢰도 0
- **SNS 삭제** → 스크린샷은 영원히 남음. 삭제 대신 정정
- **책임 회피** → 빠른 사과와 재발 방지가 장기 신뢰에 유리
- **침묵** → 정보 공백은 루머로 채워짐

## 콘텐츠 PR & 소셜미디어

### LinkedIn 기업 계정 콘텐츠 전략

**콘텐츠 믹스 (70-20-10 법칙):**
- 70%: 업계 인사이트, 유용한 정보 (교육형)
- 20%: 회사 성과, 팀 문화 (브랜드형)
- 10%: 채용, 제품 홍보 (프로모션형)

**최적 포스팅 시간:** 화-목 오전 8-10시, 오후 12-2시

**LinkedIn 알고리즘 최적화:**
- 첫 60-90분 내 댓글/공감 수 → 도달률 결정
- 외부 링크는 댓글에 → 알고리즘 페널티 방지
- 3-5개 해시태그 (업계 + 주제 + 브랜드)
- 700자 이상 본문 + 이미지/영상 조합 최적

### 임원 퍼스널 브랜딩

**CEO 소셜 포지셔닝 원칙:**
1. 한 가지 핵심 주제 영역 선점 (업계 권위자)
2. 회사 이야기보다 업계 관점 비중 높이기 (8:2)
3. 실패와 배움 공유 → 신뢰성 구축
4. 다른 사람 콘텐츠 진정성 있게 인용/공유
5. 주 2-3회 일관성 있는 게시 리듬 유지

## 소트 리더십 프로그램 (Thought Leadership)

### 주제 선정 프레임워크

소트 리더십은 회사 홍보가 아니라 업계 관점 제공이 핵심이다. 주제 선정 시 아래 3가지 교차점을 찾는다:

```
[회사의 독보적 데이터/경험]
        ↓
[업계가 아직 충분히 다루지 않은 질문]
        ↓
[타겟 독자(기자, 투자자, 고객)가 알고 싶어하는 것]
        ↓
소트 리더십 주제
```

**주제 평가 기준 (각 5점 만점):**

| 기준 | 설명 |
|---|---|
| 독창성 | 이미 많이 다뤄진 주제인가? (낮을수록 좋음) |
| 데이터 뒷받침 | 우리만 가진 수치/사례가 있는가? |
| 시의성 | 지금 업계 화두와 연결되는가? |
| 대변인 적합성 | 이 주제를 말할 자격이 있는 임원이 있는가? |
| 미디어 관심도 | 기자들이 다루고 싶어할 앵글인가? |

### 콘텐츠 캘린더 템플릿 (분기별)

```
Q[N] 소트 리더십 캘린더

핵심 주제: [CUSTOMIZE — 분기 핵심 메시지]
대변인: [CUSTOMIZE — 임원 이름/직책]

월 1:
  - 기고문: [매체명] — "[제목 초안]"
  - 팟캐스트 출연: [프로그램명]
  - LinkedIn 시리즈: [주제] 3편

월 2:
  - 컨퍼런스 발표: [행사명] — "[세션 제목]"
  - 인터뷰: [기자명/매체]
  - 웨비나 주최: "[주제]"

월 3:
  - 리포트 발행: "[보고서 제목]" (데이터 기반)
  - 미디어 라운드테이블
  - 분기 성과 측정 및 다음 분기 계획
```

### 스피커 배치 전략

**국내 주요 컨퍼런스 타겟 목록:**

| 행사 | 시기 | 타겟 세션 유형 | 제출 마감 |
|---|---|---|---|
| 스타트업 얼라이언스 행사 | 연중 | 창업/성장 스토리 | 행사 6주 전 |
| 코리아 스타트업 포럼 | 상반기 | 정책/생태계 | 행사 8주 전 |
| 넥스트라이즈 | 6월 | 글로벌 확장 | 행사 10주 전 |
| 디지털 이노베이션 서밋 | 하반기 | 기술/AI | 행사 8주 전 |
| [CUSTOMIZE — 업계 특화 행사] | [시기] | [세션 유형] | [마감] |

**세션 제안서 (Speaker Proposal) 구조:**
```
제목: [청중이 얻어갈 것을 중심으로]
요약: [150자 이내 — 왜 이 세션이 필요한가]
학습 목표: 3가지 구체적 takeaway
대변인 소개: [직책 + 관련 경험 + 신뢰도 근거]
세션 형식: [강연 / 패널 / 워크숍]
```

## 이벤트 PR 플레이북

### 3단계 이벤트 PR 구조

**사전 (D-4주 ~ D-1일):**
- [ ] 행사 예고 보도자료 배포 (D-3주)
- [ ] 주요 기자 초청 (독점 사전 브리핑 제공)
- [ ] 소셜미디어 카운트다운 콘텐츠 (D-7일부터)
- [ ] 행사 해시태그 선정 및 사전 홍보
- [ ] 미디어 키트 준비 (배경 자료, 임원 사진, 로고)

**당일 (Live Coverage):**
- [ ] 실시간 소셜미디어 업데이트 담당자 지정
- [ ] 주요 발표 내용 즉시 트윗/LinkedIn 포스팅
- [ ] 기자 현장 지원 (별도 미디어 데스크 운영)
- [ ] 사진/영상 촬영 및 즉시 배포 가능 상태 유지
- [ ] 발표 직후 보도자료 배포 (엠바고 해제)

**사후 (D+1 ~ D+2주):**
- [ ] 행사 하이라이트 영상 편집 및 배포 (D+2일)
- [ ] 참석 기자 팔로업 (추가 인터뷰 기회 제공)
- [ ] 행사 리캡 블로그 포스트 발행
- [ ] 참석자 대상 설문 및 인용 수집
- [ ] 미디어 클리핑 취합 및 성과 보고

### 올핸즈 미팅 (All-Hands) 구성

**60분 올핸즈 표준 아젠다:**

| 시간 | 섹션 | 담당 | 목적 |
|---|---|---|---|
| 0-5분 | 오프닝 + 지난 올핸즈 이후 하이라이트 | CEO | 연속성 |
| 5-20분 | 비즈니스 현황 (지표, 성과) | CEO/CFO | 투명성 |

| 45-60분 | Q&A (익명 질문 포함) | CEO | 신뢰 |

## PR 측정 프레임워크 (AMEC 모델)

AVE(광고 환산 가치)는 사용하지 않는다. AMEC의 Barcelona Principles 기반 4단계 측정:

### 측정 단계별 지표

| 단계 | 정의 | 측정 지표 예시 |
|---|---|---|
| **Outputs** | PR 활동 결과물 | 보도 건수, 도달 독자 수, 소셜 공유 수 |
| **Outtakes** | 수용자 반응 | 기사 톤(긍정/중립/부정), 메시지 전달률, 인지도 변화 |
| **Outcomes** | 행동 변화 | 웹사이트 유입, 리드 생성, 채용 지원 증가 |
| **Impact** | 비즈니스 임팩트 | 매출 기여, 투자 유치, 파트너십 체결 |

### 월간 PR 성과 보고서 구조

```
[월] PR 성과 보고서

1. 핵심 성과 요약 (Executive Summary)
   - 총 보도 건수: [N]건 (전월 대비 +/-%)
   - 주요 매체 노출: [매체명] 등 [N]개
   - 소셜 도달: [N]명

2. Outputs
   - 보도자료 배포: [N]건
   - 미디어 피칭: [N]건 / 성사율: [%]
   - 기자 미팅: [N]건

3. Outtakes
   - 보도 톤 분석: 긍정 [%] / 중립 [%] / 부정 [%]
   - 핵심 메시지 전달률: [%]
   - 경쟁사 대비 노출 점유율(SOV): [%]

4. Outcomes
   - PR 기인 웹 유입: [N]명
   - 브랜드 검색량 변화: [%]

5. 다음 달 계획
   [주요 활동 3가지]
```

## 위기 시뮬레이션 훈련 (Tabletop Exercise)

### 훈련 구조 (3시간 세션)

**참가자:** CEO, CMO, 법무, PR 담당, 운영 책임자

**Phase 1: 시나리오 브리핑 (30분)**
- 훈련 목적 및 규칙 설명
- 시나리오 배경 제시 (실제 위기처럼 단계적 공개)

**Phase 2: 시나리오 전개 (90분)**

시나리오 예시:
```
시나리오 A: 데이터 유출
  T+0: 보안팀이 고객 DB 비인가 접근 탐지
  T+30분: 기자로부터 "고객 정보 유출 사실이냐" 문의 전화
  T+2시간: SNS에 유출 데이터 일부 공개
  T+4시간: 주요 고객사 계약 해지 통보

시나리오 B: 임원 스캔들
  T+0: 온라인 커뮤니티에 임원 관련 부정 게시물 등장
  T+1시간: 언론사 취재 요청
  T+3시간: 포털 실시간 검색어 등장

시나리오 C: 제품 결함
  T+0: 고객 피해 신고 접수 (SNS 확산 시작)
  T+2시간: 방송사 취재 요청
  T+6시간: 규제기관 조사 통보
```

**Phase 3: 디브리핑 (60분)**

```
디브리핑 질문:
1. 의사결정 과정에서 가장 어려웠던 점은?
2. 실제 위기 시 정보 공백을 어떻게 채울 것인가?
3. 내부 커뮤니케이션 체계에서 개선할 점은?
4. 외부 커뮤니케이션 타이밍과 메시지는 적절했는가?
5. 이번 훈련에서 발견한 프로세스 갭은?
```

**훈련 후 액션 아이템 트래킹:**
- [ ] 발견된 갭 목록화
- [ ] 담당자 및 완료 기한 지정
- [ ] 3개월 후 재점검 일정 확정

## 이해관계자 맵 (Stakeholder Mapping)

### 영향력 x 지지도 매트릭스

```
높은 지지도
    |
    |  [관리 유지]          [핵심 파트너]
    |  낮은 영향력,          높은 영향력,
    |  높은 지지도           높은 지지도
    |
    |---------------------------- 높은 영향력
    |
    |  [모니터링]           [집중 관리]
    |  낮은 영향력,          높은 영향력,
    |  낮은 지지도           낮은 지지도
    |
낮은 지지도
```

**이해관계자 분류 및 커뮤니케이션 전략:**

| 그룹 | 커뮤니케이션 빈도 | 채널 | 핵심 메시지 |
|---|---|---|---|
| 핵심 파트너 (고영향/고지지) | 주 1회 이상 | 직접 미팅, 전화 | 협력 기회, 공동 성과 |
| 집중 관리 (고영향/저지지) | 월 2회 | 1:1 미팅, 맞춤 자료 | 우려 해소, 이익 공유 |
| 관리 유지 (저영향/고지지) | 월 1회 | 뉴스레터, 이메일 | 진행 상황 공유 |
| 모니터링 (저영향/저지지) | 분기 1회 | 공개 채널 | 일반 정보 |

## 임원 미디어 트레이닝 가이드

### 인터뷰 전 준비 체크리스트

- [ ] 기자 이름, 소속, 최근 기사 3개 확인
- [ ] 핵심 메시지 3개 준비 (각 30초 이내로 말할 수 있게)
- [ ] 예상 어려운 질문 10개 + 답변 준비
- [ ] 금지 발언 목록 확인 (법무 검토 필요 사항)
- [ ] 인터뷰 형식 확인 (전화/대면/이메일/영상)

### 브리징 기법 (Bridging Techniques)

어려운 질문을 핵심 메시지로 전환하는 기술:

| 상황 | 브리징 문구 | 예시 |
|---|---|---|
| 모르는 질문 | "확인 후 답변드리겠습니다. 제가 말씀드릴 수 있는 것은..." | "정확한 수치는 확인 후 드리겠습니다. 다만 전반적인 성장 추세는..." |
| 부정적 전제 질문 | "그 부분도 중요하지만, 더 중요한 것은..." | "경쟁이 치열한 것은 사실입니다. 하지만 우리가 집중하는 것은..." |
| 가정 질문 | "가정에 대해 답하기보다, 현재 사실은..." | "만약이라는 가정보다, 지금 우리가 하고 있는 것은..." |
| 함정 질문 | "그렇게 단순하지 않습니다. 실제로는..." | "예/아니오로 답하기 어렵습니다. 맥락을 설명드리면..." |

### 사운드바이트 작성 원칙

좋은 사운드바이트는 15-20초 안에 말할 수 있고, 인용하기 쉬운 문장이다.

```
나쁜 예: "저희 회사는 다양한 기술적 혁신을 통해 고객들에게 
         더 나은 서비스를 제공하기 위해 노력하고 있습니다."

좋은 예: "우리는 고객이 [문제]를 해결하는 데 걸리는 시간을 
         [X]에서 [Y]로 줄였습니다."
```

**사운드바이트 공식:**
- 숫자 + 구체적 결과: "6개월 만에 고객 100만 명 달성"
- 대비 구조: "경쟁사가 [A]를 할 때, 우리는 [B]를 선택했다"
- 미래 비전: "5년 후 [CUSTOMIZE — 업계 변화 예측]"

### 인터뷰 중 절대 금지 사항

- **"오프 더 레코드"** 요청 — 기자에게 오프 더 레코드는 없다고 가정
- **경쟁사 직접 비판** — 자사 강점으로 대화 전환
- **추측성 발언** — "아마도", "~일 것 같습니다" 최소화
- **내부 갈등 언급** — 어떤 형태로든 외부 노출 금지
- **미발표 정보 언급** — 엠바고 전 정보 유출 주의
