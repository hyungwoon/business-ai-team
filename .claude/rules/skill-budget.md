# 스킬 디스크립션 예산 관리

> 스킬즈 리스팅에 할당되는 토큰 예산은 전체 컨텍스트의 1%.
> 예산 초과 시 커스텀 스킬부터 디스크립션이 축소되어 트리거가 죽는다.

## 핵심 제약 (Claude Code 내부)

- 스킬 리스팅 토큰 예산: 컨텍스트 윈도우의 **1%** (`SKILL_BUDGET_CONTEXT_PERCENT = 0.01`)
- 개별 스킬 디스크립션 최대: **250자**
- 예산 초과 시 우선순위:
  1. 커스텀 스킬 디스크립션 축소
  2. 그래도 부족하면 커스텀 스킬은 이름만 표시
  3. Anthropic 공식 번들 스킬은 항상 전체 디스크립션 유지

## 디스크립션 작성 규칙

### 250자 이내 필수

모든 `.claude/skills/*/SKILL.md`의 description 필드:
- **250자 이내**로 작성
- 핵심 트리거 키워드를 **앞쪽에 배치**
- "Use when" 패턴으로 발동 조건 명확히 명시
- 불필요한 수식어/설명 제거

### Do & Don't

| Do | Don't |
|----|-------|
| "데이터 분석. SQL 쿼리, 시각화, 통계. 데이터 조회 시 사용." | "이 스킬은 다양한 데이터 분석 작업을 수행할 수 있는 종합적인 도구입니다. SQL 쿼리 작성, 데이터 시각화, 통계 분석 등을 포함합니다." |
| 60자: 핵심 키워드 + 발동 조건 | 200자+: 장황한 설명 |

### 트리거 키워드 우선순위

디스크립션 앞쪽에 배치해야 할 것:
1. **도메인 키워드**: 마케팅, 재무, 법률, 제품 등
2. **액션 키워드**: 분석, 작성, 검토, 기획 등
3. **발동 조건**: "~ 요청 시 사용", "~ 필요할 때"

## 스킬 포트폴리오 관리

### 스킬 수 제한 가이드라인

- **권장**: 핵심 스킬 **20개 이내** 유지
- **경고**: 30개 초과 시 뒤쪽 스킬 트리거 약화 가능
- 자주 안 쓰는 스킬은 비활성화 고려

### 스킬 유형 분류

각 스킬을 다음 유형으로 분류하여 관리:

| 유형 | 설명 | 모델 업데이트 영향 | 관리 방법 |
|------|------|-------------------|----------|
| **Capability** (능력 향상형) | Claude가 원래 잘 못하는 영역 보강 | 취약 — 모델이 좋아지면 불필요해질 수 있음 | 새 모델 출시 시 A/B 테스트로 필요성 재평가 |
| **Preference** (선호도 인코딩형) | 이미 잘하는 일을 내 방식으로 하게 만듦 | 안전 — 내 워크플로우는 모델 무관 | 충실도(Fidelity) 테스트로 절차 준수 확인 |

### /health 확장 항목

`/health` 커맨드 실행 시 다음 항목 추가 진단:

```
## 스킬 디스크립션 예산
- 총 활성 스킬 수: [N] / 권장 20 이내
- 250자 초과 디스크립션: [목록] — 축소 필요
- 트리거 키워드 누락 스킬: [목록] — "Use when" 패턴 추가 필요
- Capability 유형 스킬: [N]개 — 최근 A/B 테스트 여부 확인
- Preference 유형 스킬: [N]개 — 충실도 테스트 권장
```

## 정기 점검 주기

| 이벤트 | 점검 내용 |
|--------|----------|
| 새 Claude 모델 출시 | Capability 스킬 A/B 테스트 (스킬 有 vs 無) |
| 스킬 5개 이상 추가 시 | 총 스킬 수 확인 + 디스크립션 예산 체크 |
| 분기 1회 | 전체 스킬 포트폴리오 리뷰 — 미사용 스킬 비활성화 |

## 통합 지점

- **session-reminder.md**: 세션 시작 시 스킬 예산 경고 (30개 초과 시)
- **eng-workflow.md**: `/health` 커맨드에 예산 진단 추가
- **feedback-learning.md**: 스킬 트리거 실패 감지 시 디스크립션 최적화 제안
