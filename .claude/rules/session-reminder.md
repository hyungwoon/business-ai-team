# 세션 리마인드 시스템

> 세션 시작 시 과거 학습과 프로젝트 맥락을 자동으로 로드하여, 일관된 품질을 유지하는 메커니즘.

## 세션 시작 시 자동 실행

비즈니스 도메인 요청을 처음 받았을 때 (세션 내 첫 비즈니스 요청), 다음을 내부적으로 수행한다:

### 1단계: 선호도 로드

`knowledge/preferences.md`를 읽어 사용자 선호도를 내부적으로 적용한다.
- 사용자에게 별도 표시하지 않음
- 응답 생성 시 자동 반영

### 2단계: 프로젝트 맥락 확인

현재 작업 중인 프로젝트가 있으면:
1. `projects/[프로젝트명]/_context.md` 존재 여부 확인
2. 존재하면 → 로드하여 이전 작업 맥락 파악
3. 사용자에게: "이전 [프로젝트명] 작업을 이어가시는 건가요?" 확인 (맥락이 현재 요청과 관련 있을 때만)

### 3단계: 학습 지식 리마인드

`knowledge/_index.md`를 확인한다:

| 조건 | 행동 |
|------|------|
| 해당 도메인 학습 1건 이상 | `knowledge/[도메인].md` 내부 로드 → 응답에 자동 반영 |
| 해당 도메인 학습 2건 | 응답 후 안내: "이 도메인에 2건의 학습이 쌓여 있습니다. `/improve`로 리뷰하시겠어요?" |
| 해당 도메인 학습 3건 이상 | SKILL.md 반영 절차 자동 시작 (feedback-learning.md 규칙) |

---

## 도메인 전환 시 리마인드

한 세션 내에서 도메인이 바뀔 때:

1. 새 도메인의 `knowledge/[도메인].md` 존재 여부 확인
2. 존재하면 → 내부 로드
3. `knowledge/preferences.md`는 이미 로드됨 → 재로드 불필요

---

## 세션 종료 시 학습 정리

세션이 끝날 때 (기존 세션 종료 체크리스트에 추가):

### 학습 발생 시

이번 세션에서 `knowledge/`에 새 항목이 추가되었으면:

```
📝 이번 세션 학습 요약:
- [도메인]: [1줄 요약] (knowledge/[도메인].md에 저장됨)
- ...

학습 지식은 다음 세션부터 자동으로 반영됩니다.
```

### 학습 미발생 시

표시하지 않음 (불필요한 노이즈 방지).

---

## 리마인드 원칙

- **보이지 않는 품질 유지**: 대부분의 리마인드는 내부적으로 수행. 사용자에게 불필요한 표시를 하지 않는다.
- **필요할 때만 표시**: 리뷰 제안(2건 도달), 프로젝트 맥락 확인, 세션 종료 학습 요약만 표시한다.
- **expert-routing.md와 연동**: 이 규칙은 expert-routing.md의 4.5단계(knowledge 읽기)를 보완한다. 중복 로드하지 않는다.
- **성능 우선**: preferences.md는 세션당 1회만 로드. 도메인 knowledge는 도메인 진입 시 1회만 로드.
