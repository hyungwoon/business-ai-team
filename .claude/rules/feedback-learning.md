# 피드백 자동 학습 (RLVR)

> 대화 중 사용자의 보정 피드백을 자동 감지하여 `knowledge/` 디렉토리에 저장하는 시스템.
> 세션 종료 시 누적 학습을 SKILL.md에 반영한다.

## 경로 탐색

`knowledge/` 디렉토리 경로는 라우팅 규칙과 동일한 순서로 결정:

1. **로컬**: `knowledge/` (현재 프로젝트에 존재하면)
2. **글로벌**: `~/.claude/business-team/knowledge/` (로컬에 없으면)

이하 `KNOWLEDGE_DIR`은 위 규칙으로 결정된 실제 경로.

---

## 감지 패턴

대화 중 다음 패턴이 나타나면 학습 대상으로 판단한다:

### 정보 보정

사용자가 사실 관계를 교정할 때:

| 패턴 | 예시 |
|------|------|
| "아닌데", "틀렸", "틀린" | "아닌데, 그 법은 작년에 개정됐어" |
| "바뀌었", "변경됐" | "최근에 바뀌었는데, 이제는 20%야" |
| "사실은", "정확히는" | "정확히는 분기별이 아니라 월별 보고야" |
| "최신 기준으로는", "올해부터는" | "올해부터는 기준이 달라졌어" |
| "그게 아니라" | "그게 아니라 MAU 기준이야" |

### 실무 노하우

사용자가 실무 경험을 공유할 때:

| 패턴 | 예시 |
|------|------|
| "실무에서는", "현장에서는" | "실무에서는 이 방식은 안 써" |
| "경험상", "해보니까" | "경험상 이 방법이 훨씬 효과적이야" |
| "우리 회사에서는", "우리 팀은" | "우리 회사에서는 이 도구를 쓰고 있어" |
| "업계에서는", "시장에서는" | "업계에서는 보통 이렇게 접근해" |
| "실제로는", "현실적으로" | "실제로는 그렇게 단순하지 않아" |

### 주의/경고

사용자가 함정이나 위험을 알려줄 때:

| 패턴 | 예시 |
|------|------|
| "주의해야", "조심해야" | "이 부분은 주의해야 해, 법적 리스크가 있어" |
| "함정이", "실수하기 쉬운" | "여기 함정이 있는데, 세금 계산이 달라" |
| "놓치기 쉬운", "빠뜨리기 쉬운" | "놓치기 쉬운 건 환율 변동이야" |
| "절대 하면 안 되는", "금기" | "이 업종에서 절대 하면 안 되는 게 있어" |

### 선호도 (→ preferences.md에 저장)

사용자가 일관된 선호를 표현할 때:

| 패턴 | 예시 |
|------|------|
| "항상 이렇게", "매번" | "항상 표 형식으로 정리해줘" |
| "난 이걸 선호", "내 스타일은" | "난 짧고 핵심적인 보고서를 선호해" |
| "앞으로는", "다음부터는" | "앞으로는 영문 병기해줘" |
| "기본값으로", "디폴트로" | "기본값으로 분기별로 나눠줘" |

## 학습 절차 (감지 시 자동 실행)

### 1단계: 도메인 판별

현재 대화의 도메인을 판별한다. 선호도인 경우 `preferences.md`로 라우팅.

### 2단계: knowledge 파일에 행 추가

해당 `KNOWLEDGE_DIR/[도메인].md` 파일의 적절한 테이블(정보 보정 / 실무 노하우 / 주의사항)에 새 행을 추가한다.

**파일 자동 생성**: 도메인 파일이 존재하지 않으면 다음 템플릿으로 자동 생성한다:

```markdown
# [도메인] 학습 지식

## 정보 보정

| # | 날짜 | 기존 정보 | 보정 내용 | 출처 |
|----|------|---------|---------|------|

## 실무 노하우

| # | 날짜 | 노하우 | 적용 맥락 |
|----|------|--------|---------|

## 주의사항

| # | 날짜 | 주의사항 | 위험/영향 |
|----|------|---------|---------|
```

**행 형식:**

정보 보정:
```
| [번호] | [오늘 날짜] | [기존 정보 요약] | [보정된 내용] | [사용자 언급 출처] |
```

실무 노하우:
```
| [번호] | [오늘 날짜] | [노하우 내용] | [적용 맥락] |
```

주의사항:
```
| [번호] | [오늘 날짜] | [주의사항 내용] | [위험/영향] |
```

### 3단계: _index.md 카운트 갱신

`KNOWLEDGE_DIR/_index.md`의 해당 도메인 카운트를 +1 업데이트한다.

### 4단계: 학습 반영 표시

응답 끝에 다음 형식으로 표시한다:

```
> 학습 반영: [1줄 요약] → knowledge/[도메인].md
```

예시:
```
> 학습 반영: "마케팅 예산 배분 비율 50:30:20 → 40:40:20으로 보정" → knowledge/marketing.md
```

## 세션 종료 시 SKILL.md 반영 (자동)

세션 종료 체크리스트 실행 시, 다음 조건을 만족하면 SKILL.md에 자동 반영:

### 반영 조건

- 동일 스킬(에이전트 도메인)에 대해 **3건 이상** 학습이 누적
- 누적은 `_index.md`의 합계 기준 (현재 세션 + 이전 세션 합산)

### 반영 방법

1. 해당 `KNOWLEDGE_DIR/[도메인].md`에서 미반영 항목 확인
2. 관련 `plugins/[플러그인]/skills/[스킬]/SKILL.md` 읽기
3. SKILL.md 하단에 `## 실무 보정 사항` 섹션 추가 (이미 있으면 기존 섹션에 추가)
4. 학습된 내용을 간결하게 요약하여 반영
5. 반영 완료된 항목은 `knowledge/[도메인].md`에서 삭제하지 않고 유지 (이력 보존)

### 반영 형식

```markdown
## 실무 보정 사항

> 사용자 피드백으로 학습된 실무 지식입니다. (최종 업데이트: YYYY-MM-DD)

- [보정 내용 1]
- [보정 내용 2]
- [보정 내용 3]
```

---

## 중복 방지

- 동일한 내용이 이미 `knowledge/[도메인].md`에 존재하면 추가하지 않음
- 기존 항목과 충돌하는 새 정보는 기존 항목을 **업데이트** (행 교체, 날짜 갱신)
- SKILL.md에 이미 반영된 내용과 동일하면 스킵

---

## 수동 트리거

- `/improve` 커맨드: 조건(3건 이상)과 무관하게 전체 학습 지식을 리뷰하고 SKILL.md 반영을 제안
- 사용자가 "이거 기억해", "이건 항상 적용해" 등 명시적으로 학습을 요청하면 즉시 저장

---

## 능동적 학습 (Active Learning)

> 기존 수동 감지에 더해, 시스템이 적극적으로 학습 기회를 포착하는 메커니즘.

### 암묵적 만족도 판단

대화 흐름에서 만족/불만족을 추론한다:

| 신호 | 판단 | 행동 |
|------|------|------|
| 결과물을 그대로 사용하고 다음 요청으로 진행 | 만족 | 기록 불필요 |
| "다시 해줘", "이건 아닌데", 방향 전환 | 불만족 | 무엇이 부족했는지 분석 → knowledge 저장 |
| 같은 유형 요청을 3회 이상 반복 | 패턴 | "이 유형 요청이 반복됩니다. 기본 포맷이나 선호가 있으신가요?" 질문 |
| 결과물을 수정해서 사용 (파일 편집 관찰) | 부분 불만족 | 수정 부분 분석 → 보정 후보로 표시 |

### knowledge 파일 자동 생성

도메인별 knowledge 파일은 첫 학습 감지 시 자동 생성한다:

1. `KNOWLEDGE_DIR/[도메인].md` 파일이 존재하지 않으면
2. 다음 템플릿으로 자동 생성:

```markdown
# [도메인명] — 학습 지식

> [도메인] 도메인에서 사용자 피드백으로 학습된 지식을 기록합니다.

## 정보 보정

| # | 날짜 | 기존 정보 | 보정 내용 | 출처/근거 |
|---|------|-----------|-----------|-----------|

## 실무 노하우

| # | 날짜 | 노하우 | 적용 맥락 |
|---|------|--------|-----------|

## 주의사항

| # | 날짜 | 주의사항 | 위험/영향 |
|---|------|----------|-----------|
```

3. 행 추가 후 `_index.md` 카운트 갱신

### 반영 임계값 조정

- **기존 유지**: 3건 이상 → SKILL.md 자동 반영 (변경 없음)
- **추가 — 긴급 반영**: 사용자가 "이건 꼭 기억해", "중요한 건데", "매번 이래" 등 강조하면 → 즉시 SKILL.md 반영 후보로 표시 (건수 무관)
- **추가 — 리마인드**: 2건 도달 시 → 다음 세션에서 "이 도메인에 2건의 학습이 쌓여 있습니다. 리뷰하시겠어요?" 알림

### 교차 도메인 학습

하나의 보정이 여러 도메인에 적용될 수 있으면:

1. 주 도메인 knowledge 파일에 저장 (기존 절차)
2. 관련 도메인의 knowledge 파일에도 참조 추가:
   ```
   | [번호] | [날짜] | ↗ [주 도메인].md #[번호] 참조 | [적용 맥락] |
   ```
3. `_index.md`에서 참조 도메인 카운트도 +1
