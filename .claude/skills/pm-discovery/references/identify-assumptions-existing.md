## 가정 식별 (기존 제품)

네 가지 위험 영역에 걸쳐 위험 가정을 발굴하는 악마의 변호인 분석.

### 맥락

**$ARGUMENTS**의 기능 아이디어를 검증합니다.

사용자가 파일(디자인, PRD, 리서치)을 제공하면 먼저 읽습니다.

### 지시 사항

사용자가 제품, 목표, 시장 세그먼트, 기능 아이디어를 설명합니다. 다음 단계를 거칩니다:

1. **이 기능이 실패할 수 있는 이유에 대해 세 가지 시각에서 생각합니다**:
   - **프로덕트 매니저 시각**: 비즈니스 실행 가능성, 시장 적합성, 전략적 정렬
   - **디자이너 시각**: 사용성, 사용자 경험, 채택 장벽
   - **엔지니어 시각**: 기술 타당성, 성능, 통합 과제

2. **네 가지 위험 영역에 걸쳐 가정을 식별합니다**:
   - **가치(Value)**: 고객에게 가치를 창출할까요? 실제 문제를 해결하나요?
   - **사용성(Usability)**: 사용자가 사용 방법을 이해할 수 있을까요? 학습 곡선이 허용 가능한가요?
   - **실행 가능성(Viability)**: 마케팅, 영업, 재무, 법무가 지원할 수 있을까요?
   - **기술 타당성(Feasibility)**: 기존 기술로 구축할 수 있을까요? 통합 위험이 있나요?

3. **각 가정에 대해** 다음을 명시합니다:
   - 구체적으로 무엇이 잘못될 수 있는지
   - 얼마나 확신하는지 (높음/중간/낮음)
   - 테스트 방법 제안

단계별로 생각합니다. 철저하되 건설적으로 — 목표는 아이디어를 죽이는 것이 아니라 강화하는 것입니다.

## 가정 매핑 프레임워크

### 4-Quadrant Assumption Map (영향도 × 근거 수준)

가정을 두 축으로 분류하여 검증 우선순위를 결정합니다:

| | **근거 약함** (추측/직감) | **근거 강함** (데이터/리서치) |
|---|---|---|
| **영향도 높음** | 즉시 검증 — 실험 설계 | 진행 가능 — 모니터링 |
| **영향도 낮음** | 보류 — 우선순위 상승 시 검증 | 무시 가능 — 리소스 불필요 |

활용 시점: 가정 목록이 5개 이상일 때 검증 순서를 정하는 첫 단계로 사용합니다. 팀 워크숍에서 포스트잇으로 배치하면 합의 도출에 효과적입니다.

### 가정-실험 매핑 테이블

각 가정에 대해 최적의 검증 방법을 매핑합니다:

| 가정 | 위험 영역 | 영향도 | 근거 수준 | 검증 방법 | 성공 기준 | 기한 |
|------|-----------|--------|-----------|-----------|-----------|------|
| [가정 1] | 가치/사용성/실행가능성/기술 | 높/중/낮 | 높/중/낮 | [실험 유형] | [구체적 지표] | [기간] |
| [가정 2] | ... | ... | ... | ... | ... | ... |

활용 시점: 가정 식별 직후, 실험 설계 전에 사용합니다. 이 테이블을 채운 뒤 `prioritize-assumptions` 스킬로 우선순위를 확정합니다.

### 위험 평가 템플릿

각 위험 영역별로 구조화된 평가를 수행합니다:

**가치 위험 체크리스트**:
- [ ] 고객이 이 문제를 실제로 경험하는가? (근거: ___)
- [ ] 기존 대안 대비 의미 있는 개선인가?
- [ ] 고객이 비용을 지불할 의향이 있는가?

**사용성 위험 체크리스트**:
- [ ] 첫 사용 시 5분 내 핵심 가치를 경험할 수 있는가?
- [ ] 기존 워크플로우와 호환되는가?
- [ ] 학습 곡선이 타겟 사용자에게 적절한가?

**실행 가능성 위험 체크리스트**:
- [ ] 현재 비즈니스 모델과 양립 가능한가?
- [ ] 법규/컴플라이언스 이슈가 없는가?
- [ ] 영업/마케팅 팀이 효과적으로 전달할 수 있는가?

**기술 타당성 위험 체크리스트**:
- [ ] 현재 기술 스택으로 구현 가능한가?
- [ ] 성능/확장성 요구사항을 충족하는가?
- [ ] 외부 의존성(API, 파트너)이 안정적인가?

활용 시점: 기능 아이디어가 구체화된 후, PRD 작성 전에 팀이 함께 검토합니다. 체크가 안 된 항목이 곧 검증해야 할 가정입니다.
