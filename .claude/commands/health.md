# 프로젝트 건강도 대시보드

## 실행 절차

### 0단계: 경로 탐색

feedback-learning.md와 동일한 경로 탐색 규칙 적용:
1. 로컬 `plugins/`, `agents/`, `knowledge/` 확인
2. 없으면 글로벌 `~/.claude/business-team/` 폴백

### 1단계: 크기 분석

```
## 📊 프로젝트 크기 분석

| 영역 | 크기 | 파일 수 | 비율 |
|------|------|---------|------|
| plugins/ | [크기] | [스킬 수] | [%] |
| agents/ | [크기] | [파일 수] | [%] |
| knowledge/ | [크기] | [파일 수] | [%] |
| .claude/ | [크기] | [파일 수] | [%] |
```

**플러그인별 상세:**

각 플러그인의 크기와 스킬 수를 표시한다:
```
| 플러그인 | 크기 | 스킬 수 | 평균 스킬 크기 |
|----------|------|---------|---------------|
```

**대형 스킬 (200줄+)**: 상위 10개를 표시하고 분할 가능 여부 제안

### 2단계: 중복 분석

**이름 유사도**: 스킬 이름이 유사한 쌍을 탐지:
- 접미사만 다른 경우 (e.g., `-existing` / `-new`)
- 동의어 사용 (e.g., `competitive-analysis` / `competitor-analysis`)

**내용 유사도**: 같은 플러그인 내 SKILL.md 설명이 겹치는 경우 탐지

결과 형식:
```
## 🔍 중복 후보

| 스킬 A | 스킬 B | 유사도 | 비고 |
|--------|--------|--------|------|
```

### 3단계: 학습 현황

`knowledge/_index.md`를 읽어 표시:

```
## 📚 학습 현황

총 학습 항목: [N]건 ([M]개 도메인)
선호도: [K]건

| 도메인 | 보정 | 노하우 | 주의 | 합계 | SKILL.md 반영 |
|--------|------|--------|------|------|--------------|
```

**건강도 평가:**
- 0건 도메인 비율이 80% 이상 → "⚠️ 학습 시스템이 거의 활용되지 않고 있습니다"
- 3건+ 도메인 중 SKILL.md 미반영 → "⚠️ 반영 대기 중인 학습이 있습니다"
- 10건+ 도메인 → "✅ 활발히 학습 중입니다"

### 4단계: 구조 건강도

다음 항목을 체크한다:

```
## 🏗️ 구조 체크

- [ ] agents/ 파일 수 == plugins/ 플러그인 수 (1:1 매핑)
- [ ] 각 agent의 라우팅 테이블에 참조된 스킬이 실제 존재하는지
- [ ] knowledge/_index.md의 도메인 목록 == agents/ 파일 목록
- [ ] .claude/rules/ 규칙 파일 존재 (expert-routing, brainstorming, feedback-learning, session-reminder)
- [ ] .claude/commands/ 커맨드 파일 존재 (ask, route, team, improve, health)
```

불일치 항목은 구체적으로 표시:
```
❌ agents/design.md가 참조하는 스킬 'ux-audit'가 plugins/design/에 없습니다
```

### 5단계: 종합 점수

```
## 🎯 건강도 점수

| 항목 | 점수 | 기준 |
|------|------|------|
| 크기 효율성 | [A-F] | 대형 스킬 비율, 중복 비율 |
| 학습 활성도 | [A-F] | 학습 도메인 비율, 항목 수 |
| 구조 일관성 | [A-F] | 매핑 정확도, 파일 존재 |
| 종합 | [A-F] | 가중 평균 |
```

점수 기준:
- A: 90%+ 건강
- B: 80-89%
- C: 70-79%
- D: 60-69%
- F: 60% 미만

---

## 권장 조치

점수가 B 미만인 항목에 대해 구체적 개선 방안을 제시한다:

```
## 🔧 권장 조치

1. [우선순위: 높음] [구체적 개선 방안]
2. [우선순위: 중간] [구체적 개선 방안]
3. ...
```
