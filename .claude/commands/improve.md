# 학습 지식 리뷰 및 개선

## 실행 절차

### 0단계: 경로 탐색

다음 순서로 knowledge 경로를 결정한다:

1. **로컬 확인**: 현재 프로젝트에 `knowledge/` 폴더가 존재하는지 확인
   - 존재하면 → `KNOWLEDGE_DIR` = `knowledge/`
2. **글로벌 폴백**: 로컬에 없으면 `~/.claude/business-team/knowledge/` 존재 여부 확인
   - 존재하면 → `KNOWLEDGE_DIR` = `~/.claude/business-team/knowledge/`
3. **둘 다 없으면**: "학습 지식 디렉토리가 없습니다. knowledge/ 폴더를 확인하세요." 안내 후 종료.

### 1단계: 현황 대시보드

`KNOWLEDGE_DIR/_index.md`를 읽어 현황을 표시한다.

```
## 학습 지식 현황

| 도메인 | 정보 보정 | 실무 노하우 | 주의사항 | 합계 |
|--------|-----------|-------------|----------|------|
| ... | ... | ... | ... | ... |
| 합계 | ... | ... | ... | ... |

선호도: [N]건
```

항목이 0건이면: "아직 학습된 지식이 없습니다. 대화 중 보정 피드백을 주시면 자동으로 학습됩니다." 안내 후 종료.

### 2단계: 도메인별 상세 리뷰

1건 이상 있는 도메인을 순회하며:

1. `KNOWLEDGE_DIR/[도메인].md` 읽기
2. 각 항목을 다음 기준으로 분류:
   - **SKILL.md 반영 적합**: 범용적이고 반복 활용 가능한 지식
   - **프로젝트 특화**: 특정 프로젝트에만 해당하는 지식 (반영 불필요)
   - **검증 필요**: 정확성 재확인이 필요한 지식

3. 분류 결과를 표시:

```
### [도메인] — [N]건

**SKILL.md 반영 권장 ([M]건):**
1. [항목 요약] → 반영 대상: plugins/[플러그인]/skills/[스킬]/SKILL.md
2. ...

**프로젝트 특화 ([K]건):**
- [항목 요약] (유지)

**검증 필요 ([L]건):**
- [항목 요약] — 이유: [검증 필요 사유]
```

### 3단계: 반영 제안

SKILL.md 반영 적합 항목이 있으면:

```
## 반영 제안

다음 [M]건을 SKILL.md에 반영할까요?

1. [항목] → plugins/[플러그인]/skills/[스킬]/SKILL.md
2. [항목] → plugins/[플러그인]/skills/[스킬]/SKILL.md
...

전체 반영 / 선택 반영 / 취소
```

### 4단계: 사용자 승인 후 반영

승인된 항목에 대해:

1. 해당 SKILL.md 읽기
2. `## 실무 보정 사항` 섹션 추가/업데이트
3. 반영 완료 확인

```
## 반영 완료

- [스킬1] SKILL.md ← [N]건 반영
- [스킬2] SKILL.md ← [M]건 반영

knowledge/ 파일의 원본 항목은 이력으로 유지됩니다.
```

### 5단계: 선호도 리뷰 (preferences.md에 항목이 있을 때만)

`KNOWLEDGE_DIR/preferences.md` 읽기 → 현재 선호도 목록 표시 → 수정/삭제 제안.
