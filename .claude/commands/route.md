# 전문가 라우팅 커맨드

사용자의 비즈니스 요청: $ARGUMENTS

## 실행 절차 (순서 엄수, 생략 불가)

### 0단계: 경로 탐색 (로컬 우선, 글로벌 폴백)

다음 순서로 에이전트 경로를 결정한다:

1. **로컬 확인**: 현재 프로젝트에 `agents/` 폴더가 존재하는지 확인
   - 존재하면 → `AGENTS_DIR` = `agents/`, `PLUGINS_DIR` = `plugins/`, `KNOWLEDGE_DIR` = `knowledge/`
2. **글로벌 폴백**: 로컬에 없으면 `~/.claude/business-team/agents/` 존재 여부 확인
   - 존재하면 → `AGENTS_DIR` = `~/.claude/business-team/agents/`, `PLUGINS_DIR` = `~/.claude/business-team/plugins/`, `KNOWLEDGE_DIR` = `~/.claude/business-team/knowledge/`
3. **둘 다 없으면**: "에이전트 팀이 설정되어 있지 않습니다. business-ai-team 프로젝트에서 ./install.sh를 실행하세요." 안내 후 종료.

이후 모든 단계에서 `agents/`는 `AGENTS_DIR`, `plugins/`는 `PLUGINS_DIR`, `knowledge/`는 `KNOWLEDGE_DIR`로 대체하여 참조한다.

### 0.5단계: 기존 맥락 확인 (게이트 스킵 판정)

다음을 **모두** 만족하면 브레인스토밍 게이트(1단계)를 **자동 스킵**하고 2단계로 이동:

1. `projects/[프로젝트]/_context.md` 존재 (이미 진행 중인 프로젝트)
2. 관련 `*_요구사항_정리.md` 존재 (이전 요구사항이 정리됨)
3. 현재 요청이 이전 요구사항의 연장선 (후속 작업)

스킵 시: `> 기존 요구사항 기반으로 진행합니다. (게이트 스킵)` 표시.

**같은 프로젝트라도 전혀 다른 방향의 요청이면 게이트를 다시 실행한다.**

### 1단계: 브레인스토밍 게이트

`requirements-brainstorming.md` 규칙의 2축 매트릭스로 요청을 분류한다:

|  | 모호 | 구체적 |
|---|---|---|
| **과업** | A. 전체 브레인스토밍 | B. 경량 브레인스토밍 |
| **사실** | C. 1개 질문 후 라우팅 | D. 직접 라우팅 |

- **A**: 전체 브레인스토밍 절차 실행 (맥락 파악 → 1개씩 명확화 질문 → 접근 방식 제안 → 요구사항 저장)
- **B**: 1-2개 핵심 확인 후 라우팅
- **C**: 1개 범위 확인 질문 후 라우팅
- **D**: 바로 2단계로

탈출 조건: "바로 해줘", "알아서 해줘" 등 → 즉시 라우팅.

### 2단계: 도메인 분류

CLAUDE.md에 에이전트 매핑 테이블이 있으면 → 테이블에서 빠르게 식별.
없으면 → Glob으로 `AGENTS_DIR/*.md` 목록을 확인하고, 각 파일의 첫 줄 설명(> 인용문)을 읽어 가장 관련된 에이전트를 식별.

복합 요청이면 여러 에이전트를 식별한다.

### 3단계: 에이전트 시스템 프롬프트 읽기 (필수)

식별된 에이전트의 `.md` 파일을 `AGENTS_DIR`에서 Read 도구로 **반드시** 읽는다.

- 단일 에이전트: 해당 파일 1개 읽기
- 복합 에이전트: 주 에이전트 + 보조 에이전트 모두 읽기 (병렬 Read 사용)

**이 단계를 건너뛰면 안 된다. 기억이나 추정으로 대체 금지.**

### 4단계: 플러그인 스킬 읽기 (필수)

에이전트 .md 파일의 "플러그인 & 스킬 라우팅" 섹션에서 요청과 관련된 스킬을 확인하고, 해당 `SKILL.md`를 Read 도구로 **반드시** 읽는다.

경로 패턴: `PLUGINS_DIR/[플러그인명]/skills/[스킬명]/SKILL.md`

- 요청과 직접 관련된 스킬만 선택적으로 읽기 (모든 스킬을 읽을 필요 없음)
- 복수 스킬이 필요하면 병렬 Read 사용

### 4.5단계: 학습 지식 읽기

`KNOWLEDGE_DIR`이 존재하면:

1. `KNOWLEDGE_DIR/[도메인].md` 읽기 (해당 도메인의 학습 지식)
2. `KNOWLEDGE_DIR/preferences.md` 읽기 (공통 선호도)
3. 항목이 존재하면 → 응답 시 학습된 보정 사항을 **우선 반영**
4. 항목이 없거나 디렉토리가 없으면 → 스킵

**학습 지식은 SKILL.md보다 우선한다** (더 최신의 사용자 피드백이므로).

### 5단계: 전문가 관점 응답

읽은 에이전트 시스템 프롬프트의 원칙, 커뮤니케이션 스타일, 출력 기준을 적용하고,
읽은 SKILL.md의 베스트 프랙티스, 프레임워크, 템플릿을 활용하고,
학습 지식의 보정 사항을 우선 반영하여 응답한다.

### 6단계: 담당 표시 (필수)

응답 **맨 앞에** 반드시 다음 형식으로 표시:

```
> **담당**: [에이전트명] | 참조 스킬: [사용한 스킬명]
```

예시:
- `> **담당**: Marketing | 참조 스킬: content-creation, campaign-planning`
- `> **담당**: Research + Product | 참조 스킬: competitive-analysis, feature-spec`

## 검증 체크리스트

응답 전 확인:
- [ ] 브레인스토밍 게이트를 통과했는가? (또는 D분류/스킵 조건 해당?)
- [ ] 에이전트 .md 파일을 실제로 Read 했는가?
- [ ] 관련 SKILL.md를 실제로 Read 했는가?
- [ ] knowledge/ 학습 지식을 확인했는가?
- [ ] 응답에 `> **담당**:` 표시가 있는가?
- [ ] 에이전트의 커뮤니케이션 스타일과 출력 기준을 반영했는가?
